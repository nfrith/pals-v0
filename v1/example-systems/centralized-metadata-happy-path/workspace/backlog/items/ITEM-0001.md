# ITEM-0001

## DESCRIPTION

Refactor the backlog module so one item entity can represent different work types without forcing every type into one shared workflow.

## REQUIREMENTS

- Preserve a single backlog item identity shape for refs and search.
- Allow app work to use delivery-oriented statuses like `scoped` and `delivered`.
- Keep type-specific sections available without forcing them on research items.

## ARCHITECTURE

Represent type as a discriminator and treat app-specific structure as a variant layered on top of shared item identity and shared base sections.

- 2026-03-17: Captured the design goal for type-discriminated backlog items.
- 2026-03-17: Chose app as the first concrete variant to model.
