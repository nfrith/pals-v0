# PLB-002 GitHub OAuth2 App Authentication

## OVERVIEW

GitHub OAuth2 powers the "Sign in with GitHub" flow and grants scoped access to user repositories for our code analysis features. We use the web application flow, not the device flow.

## AUTHENTICATION

OAuth2 web application flow with PKCE. The app is registered under the organization's GitHub settings.

> The client secret must never be exposed to the browser. The token exchange happens server-side only.

Required environment variables:

```bash
GITHUB_CLIENT_ID=Iv1.abc123...
GITHUB_CLIENT_SECRET=secret_abc123...
GITHUB_REDIRECT_URI=https://app.example.com/auth/github/callback
```

The authorization flow:

1. Redirect user to `https://github.com/login/oauth/authorize` with `client_id`, `redirect_uri`, `scope`, and `state`
2. GitHub redirects back with a `code` parameter
3. Exchange the code for a token server-side:

```bash
curl -X POST https://github.com/login/oauth/access_token \
  -H "Accept: application/json" \
  -d client_id="$GITHUB_CLIENT_ID" \
  -d client_secret="$GITHUB_CLIENT_SECRET" \
  -d code="$AUTH_CODE"
```

Response:

```json
{
  "access_token": "gho_abc123...",
  "token_type": "bearer",
  "scope": "repo,read:user"
}
```

- Request only the scopes you need. `repo` is broad — prefer `public_repo` if private repo access is not required
- Tokens do not expire by default but can be revoked by the user at any time
- Store tokens encrypted at rest

## ENDPOINTS

### Get Authenticated User

```bash
curl -H "Authorization: Bearer $TOKEN" \
  https://api.github.com/user
```

Returns the user's profile including `login`, `id`, `email`, and `avatar_url`. Use `login` as the stable identifier.

### List User Repositories

```bash
curl -H "Authorization: Bearer $TOKEN" \
  "https://api.github.com/user/repos?sort=updated&per_page=30"
```

- Pagination via `Link` header — follow `rel="next"` until exhausted
- Use `sort=updated` to show the most relevant repos first
- Filter with `type=owner` to exclude forks and org repos if needed

### Get Repository Contents

```bash
curl -H "Authorization: Bearer $TOKEN" \
  "https://api.github.com/repos/{owner}/{repo}/contents/{path}"
```

- Returns base64-encoded file content for files under 1MB
- For larger files, use the Git Blobs API
- Directory listings return an array of entries with `type`, `name`, `path`, and `sha`

## ERROR_HANDLING

GitHub uses standard HTTP status codes with a JSON error body.

- `401 Unauthorized` — token expired, revoked, or missing. Prompt re-authentication
- `403 Forbidden` — rate limit exceeded or insufficient scope. Check `X-RateLimit-Remaining` header
- `404 Not Found` — resource does not exist or token lacks access. Do not distinguish these for security reasons
- `422 Unprocessable Entity` — validation failed. Check the `errors` array for field-level details

> GitHub's rate limit is 5000 requests/hour for authenticated requests. The `X-RateLimit-Reset` header gives you the UTC epoch time when the limit resets. Never poll — use conditional requests with `If-None-Match` to avoid burning quota.

```bash
curl -H "Authorization: Bearer $TOKEN" \
  -H "If-None-Match: \"abc123\"" \
  https://api.github.com/user/repos
# Returns 304 Not Modified if nothing changed — does not count against rate limit
```
