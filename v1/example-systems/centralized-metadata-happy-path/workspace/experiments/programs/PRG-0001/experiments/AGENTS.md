Experiments under this program. Each lives in `EXP-NNNN/EXP-NNNN.md` with runs nested below in `EXP-NNNN/runs/`.

## Working Here

DESIGN is the experiment's contract — what you're testing and how. Write it before the first run, not after. Changing the design mid-experiment means you're running a different experiment. Start a new one.

METRICS are what you'll actually measure during runs. Be specific. If a metric isn't in this list, it shouldn't show up in run observations as a primary finding.
