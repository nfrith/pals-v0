# REL-002

## CHANGES

Patch release fixing the certificate validation issue from INC-002. Adds a certificate expiry pre-check on startup and a graceful degradation path for brief signing service outages.

- Added startup check that logs a warning if the TLS certificate expires within 14 days
- Added in-memory token cache with 5-minute TTL — allows the service to validate recently-seen tokens during brief KMS or signing service outages
- Fixed log format for TLS errors to include the certificate subject and expiry date

```go
// Certificate expiry check on startup
func checkCertExpiry(certPath string) error {
    cert, err := tls.LoadX509KeyPair(certPath, keyPath)
    if err != nil {
        return fmt.Errorf("failed to load certificate: %w", err)
    }
    leaf, err := x509.ParseCertificate(cert.Certificate[0])
    if err != nil {
        return fmt.Errorf("failed to parse certificate: %w", err)
    }
    daysUntilExpiry := time.Until(leaf.NotAfter).Hours() / 24
    if daysUntilExpiry < 14 {
        log.Warn("certificate expires soon",
            "days_remaining", int(daysUntilExpiry),
            "not_after", leaf.NotAfter,
        )
    }
    return nil
}
```

## ROLLBACK_PLAN

> This release is safe to roll back. The token cache is purely additive — removing it returns to the previous behavior of real-time validation for every request.

1. Revert to the previous image:

```bash
kubectl set image deployment/auth-service \
  auth-service=registry.example.com/auth-service:1.8.2 \
  -n production
```

2. Verify rollback:

```bash
kubectl rollout status deployment/auth-service -n production
```

3. Confirm the startup certificate check log line is absent (it was added in this release)

## VERIFICATION

1. Check running version:

```bash
kubectl get pods -l app=auth-service -o jsonpath='{.items[*].spec.containers[0].image}'
```

2. Verify certificate check ran on startup:

```bash
kubectl logs -l app=auth-service --since=10m | grep "certificate"
```

3. Verify token cache is operational:

```bash
curl -f http://localhost:8080/debug/vars | jq '.token_cache_hit_rate'
```
