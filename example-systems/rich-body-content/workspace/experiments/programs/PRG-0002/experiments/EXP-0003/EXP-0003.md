# Entity Density Stress Test

## DESIGN

Compare structured compression vs. naive truncation across conversations with varying entity density to test whether structured compression's advantage scales with complexity.

Three density tiers:
- **Low density**: 2-3 entities, linear conversational flow, minimal cross-references
- **Medium density**: 5-8 entities, some cross-references between entities
- **High density**: 10+ entities with complex interdependencies and overlapping relationships

Each tier runs both the structured compression method and a naive truncation baseline against equivalent conversational inputs. Runs should use controlled conversation transcripts so that the only variable is the compression method applied.

## METRICS

- Recall accuracy by density tier (% of facts correctly recalled after compression)
- Entity relationship preservation rate (% of connections between entities correctly maintained post-compression)
- Contradiction rate by density tier (frequency of statements that contradict prior decisions or facts)
- Compression ratio (retained signal per token - useful information preserved relative to compressed size)
