# people/persons/

Individual person records live here.

## Creating People

Check existing files before assigning an ID. Collisions break everything downstream. `display_name` is how the person prefers to be addressed — get it right.

Tags are for roles and capabilities (`engineering`, `design`, `ops`), not project assignments. A person tagged `engineering` is an engineer whether or not they're on the current sprint.

## What Goes Where

- PROFILE: who they are, their role, why they're in the system. Stable over months.
- CONTACT: communication channels. Never credentials. If you're tempted to put an API key here, stop.
- NOTES: optional. Collaboration style, timezone, anything useful for working with them.
