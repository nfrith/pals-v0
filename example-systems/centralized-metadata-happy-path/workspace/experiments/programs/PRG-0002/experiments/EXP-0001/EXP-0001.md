# Baseline Compression Comparison

## DESIGN

Run identical conversation sets through two branches and evaluate recall in follow-up sessions:

- **Branch A -- Structured compression**: Prior session context is compressed while preserving entity relationships, decision points, and dependency graphs. The compressed artifact is provided as context in the follow-up session.
- **Branch B -- Naive truncation**: Prior session context is truncated using a simple token-limit cutoff with no structural awareness. The truncated text is provided as context in the follow-up session.

Each branch receives the same initial conversation set. After the initial session, a follow-up session is conducted where the model must reference prior context. Both branches are evaluated against the same recall-oriented metrics.

## METRICS

- **Recall accuracy**: Percentage of prior facts correctly referenced in the follow-up session
- **Contradiction rate**: Number of decisions or statements in the follow-up that conflict with information established in the prior session
- **Re-ask rate**: Number of questions repeated in the follow-up that were already answered in the prior session
- **Subjective coherence rating**: Human-assessed score (1--5) of how coherent and contextually grounded the follow-up session feels
